### 中文版本

昨天我在整理《智算基石》第1章时，回顾了这二十年基础设施的演进历程。发现几个有趣的规律：

**1️⃣ 资源池化 → 服务化 → 智能化**

- 2000年代：每台服务器一个应用，资源利用率<15%
- 2010年代：虚拟化让资源池化，利用率提升到60%+
- 2020年代：云计算让资源服务化，按需使用
- 现在：AI让资源智能化，自动优化调度

这不是简单的技术升级，而是思维方式的跃迁。

**2️⃣ 从"拥有"到"使用"**

- 2000年代：企业买服务器、建机房，关注资产所有权
- 2010年代：企业租用云资源，关注服务质量
- 2020年代：企业使用AI能力，只关注业务价值

计算正在从"资产"变成"空气"——随处可得，按需使用。

**3️⃣ 硬件性能每提升一个数量级，架构就质变**

- 千兆→万兆网络：存储从FC转向以太网
- 万兆→25G网络：分布式存储成为可能
- CPU→GPU：计算范式从通用转向专用
- 单机→多机：高速互联网络成为刚需

每一次硬件突破，都催生新的架构范式。

**4️⃣ 复杂度不断向上层转移**

- 物理机时代：运维关注硬件故障、机房环境
- 虚拟化时代：关注虚拟机迁移、资源调度
- 云原生时代：关注容器编排、服务网格
- AI时代：关注模型训练、数据管线、成本优化

我们解决的问题越来越抽象，但创造的价值也越来越大。

### English Version

💭 From Virtualization to Intelligent Computing: 4 Patterns in 20 Years of Infrastructure Evolution

Yesterday while organizing Chapter 1 of "The Foundation of Intelligent Computing," I reviewed the evolution of infrastructure over these two decades. Here are some interesting patterns I discovered:

**1️⃣ Resource Pooling → Servitization → Intelligence**

- 2000s: One application per server, resource utilization <15%
- 2010s: Virtualization enabled resource pooling, utilization increased to 60%+
- 2020s: Cloud computing made resources servitized, on-demand usage
- Now: AI makes resources intelligent, auto-optimization

This isn't just technical upgrades—it's a paradigm shift in thinking.

**2️⃣ From "Owning" to "Using"**

- 2000s: Companies bought servers, built data centers, focused on asset ownership
- 2010s: Companies rented cloud resources, focused on service quality
- 2020s: Companies use AI capabilities, only care about business value

Computing is becoming like "air"—available everywhere, on demand.

**3️⃣ Every Order of Magnitude Improvement in Hardware Performance Triggers Architectural Transformation**

- Gigabit → 10GbE: Storage shifted from FC to Ethernet
- 10GbE → 25GbE: Distributed storage became feasible
- CPU → GPU: Computing paradigm shifted from general to specialized
- Single machine → Multi-machine: High-speed interconnect became essential

Each hardware breakthrough spawns new architectural paradigms.

**4️⃣ Complexity Continuously Shifts to Higher Layers**

- Physical machine era: Ops focused on hardware failures, data center environment
- Virtualization era: Focused on VM migration, resource scheduling
- Cloud-native era: Focused on container orchestration, service mesh
- AI era: Focused on model training, data pipelines, cost optimization

The problems we solve are becoming more abstract, but the value we create is also increasing.
